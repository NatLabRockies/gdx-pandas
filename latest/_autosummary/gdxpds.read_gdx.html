gdxpds.read_gdx.to_dataframes(gdx_file: str | os.PathLike[str], gams_dir: str | os.PathLike[str] | None = None, engine: str | Engine | None = None, symbols: list[str] | None = None) → dict[str, pd.DataFrame][source]

Primary interface for converting a GAMS GDX file to pandas DataFrames.

Parameters:

  • gdx_file (pathlib.Path or str) – Path to the GDX file to read

  • gams_dir (None or pathlib.Path or str) – optional path to GAMS directory

  • engine (None or str or gdxpds.Engine) – Which I/O engine to use. None (default) resolves via the GDXPDS_ENGINE env var, then the default engine (gams.transfer when usable, otherwise gdxcc).

  • symbols (None or list of str) – If None (default), every symbol is read. Otherwise only the named symbols are read and returned, in the given order; an unknown name raises SymbolNotFoundError and [] returns an empty dict.

Returns:

Returns a dict of Pandas DataFrames, one item for each requested symbol in the GDX file, keyed with the symbol name. For a Set or Alias, the value column holds the GAMS element text ("" for a member with no text); membership is conveyed by row presence.

Return type:

dict of str to pd.DataFrame

gdxpds.read_gdx.get_subset_relationships(gdx_file: str | os.PathLike[str], gams_dir: str | os.PathLike[str] | None = None, engine: str | Engine | None = None) → dict[str, list[str | None]][source]

Returns the domain relationships recorded in gdx_file, keyed by symbol name.

Any symbol type (Set, Parameter, Variable, Equation) can carry a domain – a Set’s domain is a subset relationship, a Parameter/Variable/Equation’s is an indexed-over relationship – and the parent named in each slot is a Set or Alias-of-Set.

Outputs a dict that maps each symbol name to a list with one entry per dimension, giving the parent Set name recorded for that dimension. A dimension whose domain is the wildcard ('*'), or for which the GDX file records no domain information at all, comes through as None. Every other dimension is reported by its recorded name verbatim, including the self-referential case where a (typically root) Set’s dimension names the Set itself.

The length of each list matches the symbol’s number of dimensions, and names appear in dimension order. The output shape matches the domains= argument of to_gdx(), so a value read here can be fed straight back in (None round-trips as the wildcard).

Parameters:

  • gdx_file (pathlib.Path or str) – Path to the GDX file to read

  • gams_dir (None or pathlib.Path or str) – optional path to GAMS directory

  • engine (None or str or gdxpds.Engine) – Which I/O engine to use (default resolves via GDXPDS_ENGINE).

Returns:

Map of symbol name to its domain. Pair this with to_dataframes() to recover the full file shape.

Return type:

dict of str to list of (str or None)

gdxpds.read_gdx.get_aliases(gdx_file: str | os.PathLike[str], gams_dir: str | os.PathLike[str] | None = None, engine: str | Engine | None = None) → dict[str, str][source]

Returns the alias relationships recorded in gdx_file, keyed by alias name.

The output shape matches the aliases= argument of to_gdx(), so a value read here can be fed straight back in. Each entry maps an alias’s name to the name of its parent symbol – typically a Set, but a chained alias parent is possible (GDX permits a chain, and the gdxcc engine preserves it on disk; gams.transfer flattens to the root Set on write). The universe set ('*') is reported verbatim for a universe alias.

Symbols that are not aliases are not included in the output.

Parameters:

  • gdx_file (pathlib.Path or str) – Path to the GDX file to read

  • gams_dir (None or pathlib.Path or str) – optional path to GAMS directory

  • engine (None or str or gdxpds.Engine) – Which I/O engine to use (default resolves via GDXPDS_ENGINE).

Returns:

Map of alias name to parent symbol name. Pair this with to_dataframes() and get_subset_relationships() to recover the full file shape.

Return type:

dict of str to str
